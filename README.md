# Furniture Store Database System

## Overview

This project focuses on the design and implementation of a relational database system for a custom furniture store using MySQL and Enhanced Entity Relationship (EER) modelling.

The system was developed to support customer management, custom furniture requests, quotations, orders, payments, inventory tracking, and product categorisation.

The project includes conceptual database modelling, SQL database implementation, sample data population, and SQL query analysis.

## Project Components

### Conceptual Database Modelling

Designed an Enhanced Entity Relationship (EER) model using Crow’s Foot notation to represent:

* entities
* relationships
* constraints
* weak entities
* associative entities
* inheritance relationships

### Relational Database Implementation

Implemented the database schema using SQL CREATE TABLE statements with:

* primary keys
* foreign keys
* NULL / NOT NULL constraints
* data validation rules
* relational integrity constraints

### Sample Data Population

Created INSERT statements to populate the database with test data for:

* customers
* staff members
* orders
* furniture items
* payments
* quotes
* product categories

### SQL Query Development

Developed SQL queries to:

* filter and sort customer records
* calculate order totals
* analyse payments
* determine outstanding balances
* generate yearly order statistics
* retrieve aggregated business information
